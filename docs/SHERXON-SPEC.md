## 2. ТЕХНОЛОГИЯ

> _Янгиланди 2026-06-30: чуқур таҳлилдан кейин (Toast/Square/Lightspeed/Clover benchmark) стек ўзгарди. Эски «Next.js + Supabase» БЕКОР — қуйидаги ўз-серверли стек кучда._

- **Frontend:** **Vite + React 19 + TypeScript SPA**, **PWA** (service worker + Dexie/IndexedDB) — **offline-first**: свет/интернет ўчса ишлайди, улангач синхрон. Tailwind v4 + shadcn/ui. Битта build → POS (3 касса) + официант телефонлари + бэк-офис.
- **API:** thin **Hono + tRPC** (end-to-end типли) — модуляр монолит, 1 процесс. REST `/api/v1/*` фақат ташқи вебхуклар (Click/Payme/фискал) учун.
- **БД:** ўз-серверли **PostgreSQL 17** (битта инстанс) + **Drizzle ORM**. Бирор cloud йўқ.
- **Auth:** **Better Auth** — 4-рақамли PIN, RBAC (директор/менежер/бозорчи/кассир/официант). ⚠️ ҳар authorization API/маълумот қатламида текширилади, клиентга ишонилмайди.
- **Offline-синхрон:** Dexie локал стор + append-only mutation outbox (UUID idemпотентлик) → улангач flush. Realtime: Postgres LISTEN/NOTIFY → 1 WS/SSE канал.
- **Эга-таҳрирланувчи:** **Refine** admin — ҳар қоида = типли Postgres қатор (меню/нарх/тех-карта/зал%/норма/штраф). Zod + Postgres CHECK ёмон save'ни блоклайди; audit_log триггер; нарх/рецепт **effective-dated, append-only**. Save → config_version → ҳамма қурилмага дарров, redeploy'сиз.
- **Ўз-хостинг:** битта `docker-compose.yml` = **caddy** (статик SPA + reverse-proxy `/api`+`/trpc`+WS) + **api** (Hono, non-root) + **postgres:17** (`wal_level=replica`, `archive_mode=on` биринчи ёзишдан олдин) + **pgBackRest** бэкап. Docker Compose (НЕ Kubernetes).
- **Сервер:** **Dell OptiPlex** — Sanoat Uquv серверхонаси, LAN **192.168.1.4**, Limonariyaга ажратилган.
- **Кирилл ўзбекча** интерфейс
- **3 компьютер-касса + ~10 официант телефони** (~80% Android / 20% iPhone — PWA ҳаммасини қоплайди)
- Кўп-филиалли архитектура (ҳар жадвалда `branch_id` 1-кундан; ҳозир 1 филиал)
- UZS бутун сон (тийинсиз), `Math.round()`
- Электрон тарози — ⚠️ ҳали олинмаган: 1-босқич **қўлда вазн**, кейин интеграция

## 4. МАЪЛУМОТ МОДЕЛИ (асосий жадваллар)

|---|---|---|

### 4.2 Омбор / Локация ва Қолдиқ
- **Локация:** Муздаткич (1) · **Маринад** · **Витрина** (сих 3-босқич учун)
- **Қолдиқ ўзи ҳисобланади** (ҳаракатлардан), қўлда ўзгартириб бўлмайди
- Бирлик: кг / дона / л / шт

### 4.3 Ҳаракат (Inventory Movement) — ҳамма нарса шу ерда
| Тур | Изоҳ |
|---|---|
| Харид (кирим) | Бозорчи — нарх+геолокация+изоҳ |
| Обвалка | Туша − → қисмлар + (±5% баланс) |
| Ишлаб чиқариш | Хом − → ярим-тайёр + (мариновка/сихлаш/фарш) |
| Списание (сотув) | Чек ёпилганда — тех-карта бўйича |
| Инвентаризация тузатиш | Фарқ + сабаб + директор тасдиқ |
| Йўқотиш/бузилди | Сабаб + директор |
| Локация кўчириш | Маринад → витрина |

### 4.4 Заказ / Чек / Тўлов
- **Заказ:** официант очади (стол, зал тури), қаторлар (таом+сони+изоҳ), станцияга чоп. Очган официант ёпади.
- **Чек ёпилганда:** списание + хизмат ҳақи **(ҳар зал: Асосий 10% · Катта 10% · Терраса 15% · Собой 0%)** + атрибуция (официант+кассир)
- **Тўлов:** кўп-тўловли (нал/Click/Payme/қарз) — Click/Payme қўлда тасдиқ
- **Ҳисоб бўлиш** (split), **пре-чек**, **банкет** (брон+аванс)

### 4.5 Қарз (Customer ledger)
Манфий қолдиқ (бир мижоз = бир running balance) + ўзгармас журнал. Ҳар қарзга чек+изоҳ бириктирилади. SMS эслатма + рейтинг 🟢🟡🔴.

### 4.6 Молия / Касса
- Размен 50к · кассадан чиқим 3 тур (харид тўлови / харажат / эга олди) · камомад
- OPEX (126млн/ой рўйхат) · солиқ 4% (картадан авто)

### 4.7 Вазифа назорати + Штраф
- Вазифа: дедлайн / даврий / воқеа+таймер · чек-лист · жонли камера расм · админ тасдиқ
- Штраф журнали (per-task сумма; зинапоясимон ой нолланади ёки фикс)

### 4.8 Аномалия / Тешик флаглари

### 4.9 Сих/витрина аниқлик (раунд 11)
- **Ҳамма шашлик тури витринада алоҳида дона.** Меню таоми ↔ витрина сихи **1:1**.
- **Премиум кесим → ёзилганидек таомга** (Вағора→Вағури в.ҳ.). Ҳар кесим ўз таомига.
- **Рулет ва Вафли ҳам СИХга урилади** → витринада дона:
  - **Вафли:** 140 г/сих · 1 паднос = 6.1 кг (фарш 6.1кг + лаваш/хамир 6 шт) → ~43 сих

  - Нарх: менюдан (сих/порция нархи)

### 4.10 ТЕХ-КАРТА манбаси — `docs/texkarta.json` (импорт)
**26 иссиқ таом + 12 салат + Фарш (ишлаб чиқариш)** — тўлиқ рецепт (ингредиент+грамм+`stock_hint`). Бу — асосий тех-карта сиди. Муҳим:
- **Мариновка ўсиши (gain %)** ҳар таомда: Кусковой мол +13% · қўй +15% · Вағури +30% · Авғонча +30% · Уйғурча +200г · Сарёғли/Жиз филе +20%. → сих грамми/таннарх шуни ҳисобга олади.
- **Шашлик Марварид:** 1 партия = **50 сих** ← мол гўшт 1кг + думба 700г + **фарш 5кг** (асосан фарш).

- `aliases` (норм.) ва `ingredient_usage` (частота) ҳам бор.

### 4.12 ҚУРИЛМАЛАР ва ОФИЦИАНТ МОБИЛ (Clopos жонли таҳлилидан, 2026-06-25)

**Clopos ҳозир қандай ишлайди (8 қурилма):**
- **3 десктоп POS** (касса): Terminal 1 (11111), T2 (22222), T3 (33333) — паролар **кетма-кет, очиқ** (хавфли!)
- **5 официант телефони**: Telefon 1–5 — тип «Официант», ҳаммаси **Terminal 1 га боғланган** (slave), ҳар бирида PIN
- Флаглар: статус · уведомление · онлайн (телефонлар кўпи **офлайн** ✗)
- Оқим: телефонда заказ → Terminal 1 синхрон → кухня чоп → десктопда кассир ёпади

**Камчиликлар:** ① парол **қурилмага** боғланган, одамга эмас (10 официант, 5 телефон — навбат) ② парол сохта (11111) ③ офлайн бўлса заказ муаммо ④ битта терминалга боғлиқ.

**🚀 БИЗ ҚЎШАМИЗ (яхшилаш):**
1. **PIN ҳар ОФИЦИАНТга** (қурилмага эмас) → исталган телефонга ўз PINи билан киради → заказ **ўша официантга** ёзилади (навбат осон, 10/5)
2. **Offline-first** → свет/интернет ўчса ҳам заказ олинади, улангач синхрон (Clopos'да йўқ)
3. **Жонли витрина светофори телефонда** → таом тугаганини кўради (🟢🟡🔴), тугаганни сотмайди
4. **Заказ ҳолати + push** → юборилди→пишяпти→тайёр→берилди; тайёр бўлса push → тез хизмат
5. **Стол харитаси телефонда** → зал планини кўриб, столга босиб заказ очади (Clopos'да координата бор)
6. **Стол йиғиштириш таймери** телефонда (чек ёпилгач 10 мин → 40к)
7. **Хавфсиз PIN** (хэшланган, кетма-кет эмас, директор тиклайди)
8. **Официант ўз статистикаси** телефонда (бугун қанча сотди — мотивация; директор лидербордни кўради: Асадбек 17.3М...)
9. **Ақлли кухня роутинг** → таом ўз станциясига (мангал/ошхона/салат/бар) авто чоп

### 4.11 Сих/витрина — раунд 11 жавоблар

## 5. WORKFLOW — 6 ОҚИМ (тўлиқ: `docs/WORKFLOW.md`)
1. **Харид + Обвалка:** Бозорчи (геолокация) → 2 повар обвалка → менежер вазн ёзади → ±5% баланс → аномалия → SKLAD 1.
2. **Мариновка / Ишлаб чиқариш:** хом гўшт → маринад → сих (витрина) · фарш/рулет/вафли (заготовка) — шашликчи қилади.
3. **Сотув:** официант мобил POS → станция чоп → кассир (кўп-тўлов) → **чек ёпилганда списание** + хизмат ҳақи (зал бўйича) + атрибуция.
4. **Қарз:** ledger + чек + изоҳ + SMS + рейтинг.
5. **Кун ёпилиши (06:00):** кассир нақд санайди (размен 50к − кассадан чиқим 3 тур − возврат) → директор тасдиқ. Инвентаризация = **эрталаб** (менежер).
6. **Директор назорати:** 4 рақам + остаток + тешиклар + push.

## 6. ОБВАЛКА + ТАННАРХ МОДЕЛИ
Туша → қисмлар (тарози авто). Туша нархи **сотиладиган гўшт қисмлари**га тарқалади. **Суяк/чарви/брак = cost 0**. Йўқотиш % ҳисобга олинади → реал 1кг таннарх. Мариновка ўсиши (texkarta: +13–30%) ҳисобга олинади. Норма тарихдан (динамик).

## 7. СИХ / ВИТРИНА — 3 БОСҚИЧ
Хом гўшт(кг) → маринад(кг) → витрина(дона) → сотув. **«Ҳисобланган сих ↔ қўлда киритилган сих»** солиштируви (грамм/йўқотиш назорати). FIFO + эски огоҳ. Меню таом ↔ витрина сих 1:1.
