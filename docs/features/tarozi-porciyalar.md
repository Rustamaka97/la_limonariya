# Тарози / Продажи по порциям (тортиб/порция сотув) — backend + frontend spec

> Гўшт/салат/думба каби маҳсулотларни **оғирлик (грамм) бўйича** сотиш — тарозидан ўқиб нарх авто-ҳисоблаш. La Limonariya'да майдон бор, темир йўқ. Приоритет: **P2**.

## 1. CloPOS'да (жонли — тест қилинди)
- Заказ экрани тепасида **«Продажи по порциям»** тугма → **toggle**: босилса «**Запрос порций активирован!**».
- Ёқилганда: таом қўшганда **порция/оғирлик сўрайди** (тарозидан авто ёки қўлда). soldByWeight таомлар учун.
- Темир: тарози (serialport/COM) — CloPOS `inject.js` `scaleWeight.start/onData` (§ CLOPOS-KOCHIRISH-SHERXON §1.1).

## 2. La Limonariya — ҲОЗИР
- **`products.soldByWeight`** майдон **БОР** (тайёр). `computeOrderStockMoves`да soldByWeight таомлар списаниеда skip қилинади (граммда алоҳида).
- **ЙЎҚ:** POS'да «тортиш/порция» оқими, тарози темир уланиши.

## 3. BACKEND spec
- **Тарози ўқиш:** мост-агент (§ stansiya-pechati-bridge) `serialport` билан COM-портдан оғирлик ўқийди (CloPOS `scaleWeight/reader.js` каби — хом байт → веб парслайди). Веб'га WS/HTTP орқали узатади.
- **`pos.addItem` кенгайтириш:** soldByWeight таом учун `weightG` (грамм) қабул қилиш → `orderItems.qty = weightG`, нарх = `(weightG/1000) × price_per_kg`.
- Списание: граммда (мавжуд рецепт/carcass мантиғи билан мос).

## 4. FRONTEND spec (`Pos.tsx`)
- **«Продажи по порциям» toggle** (тепа): ёқилса — soldByWeight таом босилганда **оғирлик modal** (тарозидан **авто** ўқиш ёки **қўлда** киритиш) → нарх авто-ҳисобланади (оғирлик × нарх/кг).
- Тарози ҳолати (уланган/йўқ) — Статус панелида (§ top-bar).

## 5. Приоритет / боғлиқлик
- **P2** — гўшт/салат/думба тортиб сотиш (реал ресторонда керак).
- **Станция печати мост-агенти** билан бирга (иккови ҳам serialport/темир) — § stansiya-pechati-bridge. `soldByWeight` майдон тайёр.
