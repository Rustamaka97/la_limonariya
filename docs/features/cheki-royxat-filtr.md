# Чеки — чеклар рўйхати + фильтрлар (открытые/закрытые/офлайн) — backend + frontend spec

> CloPOS «Чеки» — барча чекларни битта жойда: **табллар** (Открытые/Закрытые/Офлайн архив) + **бой фильтрлар** (тип/официант/зал/стол/курьер). La Limonariya'да қисман (закрытые=ChekQidirish, открытые=FloorView). Приоритет: **P2** (қулайлик).

## 1. CloPOS'да (жонли кўрилди)
- **Табллар:** Открытые (очиқ) · Закрытые (ёпилган, тўлов усули билан) · **Офлайн архив** (интернетсиз тузилган).
- **Фильтрлар:** Id (чек№) · **Тип продажи** (На месте/Доставка/С собой) · **Сотрудник** (официант) · **Зал** · **Стол** · **Курьер**.
- Юқорида: **Количество чеков: N** + **Сумма: X** (жами). Ҳар чек: `#№ · вақт · зал•стол · официант · сумма · тўлов усули`; ⚠️ узоқ очиқ чекда warning; «Пречек» badge.

## 2. La Limonariya — ҲОЗИР
- **Закрытые:** `ChekQidirish.tsx` + `finance.searchOrders({from,to,query})` — **сана оралиғи + чек№/стол** бўйича. Чек босилса → `OrderDetail` (receipt + возврат).
- **Открытые:** `Pos.tsx` `FloorView` + `pos.openOrders` — **floor харита** (флат рўйхат эмас).
- **ЙЎҚ:** официант/зал/тип/курьер фильтрлари, жами сумма/сон, «Офлайн архив» таб, очиқ чекларни флат фильтрланадиган рўйхат.

## 3. BACKEND spec (кенгайтириш — `router.ts`)
- **`finance.searchOrders` кенгайтириш:** мавжуд `{from,to,query}`га қўшиш — `waiterId?, hallId?, saleType?('zal'/'soboy'/...), status?('closed'/'open'/'cancelled')`. Натижага **жами**: `{count, sumTotal}`.
  - Хавфсизлик: **иерархия** — official фақат ўзиникини (мавжуд `assertOrderAccess` фалсафаси; `searchOrders`'га `role==='waiter' ? waiterId=ctx.user.id` фильтр қўшиш).
- **Курьер/Тип продажи** — «Доставка» feature'га боғлиқ (заказ турига `courierId`/`type` қўшилгач ишлайди).
- **Офлайн архив** — алоҳида «оффлайн режим» feature (SQLite; бу MD доирасидан ташқари).

## 4. FRONTEND spec (`ChekQidirish.tsx` кенгайтириш ёки янги `Cheki.tsx`)
- **Табллар:** Открытые · Закрытые (Офлайн — кейин).
- **Фильтр чиплар:** Официант · Зал · Стол · Тип продажи (+ курьер — доставка бўлса). Мавжуд сана+қидирув сақланади.
- **Юқорида жами:** «N чек · X so'm».
- Чек босилса → мавжуд `OrderDetail` (receipt + возврат) — тайёр, қайта ишлатилади.
- **Открытые таб** — очиқ заказларни флат рўйхат (FloorView'га муқобил кўриниш; иккови ҳам қолиши мумкin).

## 5. Хулоса Шерхонга
- Асосан **`searchOrders` + `ChekQidirish` кенгайтириш** (фильтрлар + жами + Открытые таб). Нолдан эмас.
- **Курьер/Тип** — «Доставка» feature билан келади. **Офлайн архив** — алоҳида «оффлайн» feature.
- Иерархия: официант фақат ўзиникини (searchOrders'га фильтр).
