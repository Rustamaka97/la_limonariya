# Клиенты — мижоз CRM + ҳамён (лоялти) — backend + frontend spec

> CloPOS'да **тўлиқ мижоз тизими**: профиль + баланс(депозит) + бонус(кэшбэк) + гуруҳ(скидка%) + статистика + харид тарихи. **La Limonariya'да ЙЎҚ** (фақат исм+телефон, қарз учун). Бу — катта янги функция, босқичма-босқич.

## 1. CloPOS'да (жонли, профиль ичи кўрилди)
- **Профиль:** Рустам (ID: 1) + телефон + **ГРУППА** (масалан «0%» — гуруҳ скидка фоизи, VIP tier).
- **Ҳамён:** **Баланс** (депозит) · **Бонус** (кэшбэк) · **Скидка**.
- **Статистика:** Общая сумма расходов (75,900) · Общая скидка · Общий бонус · **Всего чеков (6)**.
- **Харид тарихи:** ҳар чек — `Чек# · Чек закрыт · Промежуточный итог · Способ оплаты`. Тўлов усули орасида **«Баланс Клиента»** (депозитдан тўлаш!).
- **«Создать операцию»:** **Доход / Расход** → мижоз балансига пул қўшиш/ечиш + **Подтвердить**.
- **«Редактировать профиль»**.
- **«Создать профиль клиента» формаси (жонли — тўлиқ майдонлар):**
  - **Полное имя** · **Карта** (RFID hex, ex: 564as8AFF845f) · **Мобильный** (+998, бир нечта «+») · **Группа клиентов** (лоялти гуруҳ — скидка/бонус%)
  - **Пол** · **Дата рождения** (туғилган кун — CRM/акция) · **E-Почта** (маркетинг)
  - **Адреса** (бир нечта манзил — ★ **доставка учун**) · **Описание** (изоҳ)
  - **Сканировать QR-код кошелька** (рақамли ҳамён/лоялти боғлаш)
- Заказ экранида **«Добавить клиента»** → мижоз пикери (Поиск → **Применить** чекка боғлаш) — тарих + бонус + баланс тўлов.

## 2. La Limonariya — ҲОЗИР
- `customers` жадвал: фақат `name, phone` (қарзда мажбурий). `finance.customers.search/create`. Заказ ёпишда қарз учун `customerId` боғланади.
- **ЙЎҚ:** баланс, бонус, гуруҳ, скидка%, статистика, операция, «баланс» тўлов усули, харид тарихи кўриниши.

## 3. BACKEND spec (янги — `apps/api/src`)

### 3.1. Схема (Drizzle, `schema.ts`)
- **`customers` кенгайтириш (профиль майдонлари):** `+ cardId text unique?` (RFID) · `+ groupId uuid → customer_groups` · `+ gender text?` · `+ birthDate date?` (туғилган кун акцияси) · `+ email text?` · `+ note text?` · `+ walletQr text?` (ҳамён QR). (`name, phone` бор.)
- **`customer_addresses` (янги — доставка):** `id, customerId → customers, label, address, lat?, lng?` (бир нечта манзил; § Доставка билан боғлиқ).
- **`customers` ҳамён:** `+ balance int default 0` (кэш эмас — ledger'дан SUM), `+ bonus int default 0` (реал қиймат ledger'да).
- **`customer_groups` (янги):** `id, name, discountPct int default 0, cashbackPct int default 0, active bool`.
- **`customer_operations` (янги, append-only ledger):** `id, customerId, type pgEnum('deposit','withdraw','bonus_add','bonus_spend'), amount int, note, performedById, createdAt`.
  - **Баланс = SUM(deposit − withdraw)**, **Бонус = SUM(bonus_add − bonus_spend)** — ustunda saqlanmaydi, ledger'дан ҳисобланади (La Limon фалсафаси — подделкага чидамли, [[clopos-handoff-progress]]).
- **`paymentMethod` enum'га `balance` қўшиш** (баланедан тўлаш; миграция + `PAY_LABEL`га «Баланс»).

### 3.2. tRPC procedures (`router.ts`)
- **`customers.get({id})`** — профиль + `balance`(ledger SUM) + `bonus` + group(discountPct/cashbackPct) + **статистика** (SUM(closed orders total), Всего чеков, Общая скидка) + **тарих** (orders where customerId, checkNo/сана/сумма/тўлов).
- **`customers.createOperation({customerId, type, amount>0, note})`** — `customer_operations`га insert. Роль: director/manager/cashier. `withdraw`/`bonus_spend`да **баланс/бонус етарлилигини текшир** (advisory-lock, `paySupplier` каби over-draw ҳимояси).
- **`customers.groups.list/create/update`** — director (скидка%/кэшбэк% созлаш).
- **`customers.update({id, phone?, groupId?})`** — профиль таҳрири.
- **`pos.close` кенгайтириш:** `balance` тўлов усулини қабул қил → атомик `customer_operations`га `withdraw` ёз (баланс етарли бўлса; акс ҳолда BAD_REQUEST). **Кэшбэк:** ёпилганда `bonus_add = round(total × group.cashbackPct/100)` (гуруҳ бўлса). **Скидка:** `group.discountPct` — subtotal'дан чегирма (§ Скидка билан боғлиқ — алоҳида feature MD).
- Иерархия: mavjud `assertOrderAccess` таъсир қилмайди; операция cashier+ (пул).

## 4. FRONTEND spec (янги — `apps/web/src`)
- **`Clienty.tsx` (янги экран, `Shell.tsx` menu'га; роль: cashier+):**
  - Мижозлар рўйхати + қидирув (исм/тел) + «Создать профиль».
  - **Профиль:** карталар — Баланс · Бонус · Гуруҳ(скидка%) · Статистика (Общая сумма расходов, Всего чеков). Тарих жадвали (чек№ · сана · сумма · тўлов). Тугмалар: **«Создать операцию»** (Доход/Расход modal → `createOperation`), **«Редактировать»**.
- **POS (`Pos.tsx`):** мавжуд **«Добавить клиента»** (қарз учун бор) — кенгайтириб, боғланган мижознинг **баланс/бонус**ини кўрсатиш. **Тўлов modal'ига «Баланс Клиента» тугма** (баланедан тўлаш; баланс кўрсатилади).
- **Гуруҳ бошқаруви** (director): скидка%/кэшбэк% созлаш (кичик CRUD).

## 5. Босқичлар (приоритет)
1. **`customer_groups` + скидка%** — лоялти tier, содда, тез қадр (мижозга % чегирма).
2. **Баланс (депозит) + «Баланс» тўлов усули** — олдиндан тўлов/банкет депозит.
3. **Бонус (кэшбэк) accrual** — ёпилганда бонус йиғилиши.
4. **Статистика/тарих кўриниши** — CRM аналитика.

## 6. Хулоса Шерхонга
- **Катта янги функция** — босқичма-босқич қил (1→4). Ҳаммасини бирдан эмас.
- **Ledger фалсафаси мажбурий:** баланс/бонус = SUM(operations), ustunda emas.
- Мавжуд `customers` + қарз оқимини **кенгайтир** (нолдан эмас). Заказдаги «Добавить клиента» — мавжуд, шунга уланади.
- Боғлиқ: **Скидка** (гуруҳ discountPct → заказ чегирмаси) — алоҳида feature MD билан мослаш.
