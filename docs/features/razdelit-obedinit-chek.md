# Разделить / Объединить чеки (split / merge bill) — backend + frontend spec

> Битта заказни бир нечта чекка **бўлиб**, ҳар бирини **алоҳида тўлаш** (банкет/катта стол — меҳмонлар алоҳида тўласа). Ва аксинча — чекларни **бирлаштириш**. La Limonariya'да ЙЎҚ. Приоритет: **P2**.

## 1. CloPOS «Разделить заказ» (жонли — тўлиқ UI кўрилди)
- Экран 3 қисм: **Основной Чек** (асосий) | **Разделить-1 / -2...** (splits) | ўнгда **split карточкалар** (➕ билан янги split; ҳар бири «N элементов · сумма»).
- **Оқим:** Основной чекдаги **позицияга босасан** → у танланган **Разделитьга ўтади**. Split'даги позицияни босса — **Основнойга қайтади**. Шундай позицияларни тақсимлайсан.
- **Ҳар split (Разделить-N):** жой/тип (🍽 «На месте: Асосий зал / 4-Банкет зал») · **🔒 Блокировка квитанции** (чекни қулфлаш) · **🖨 Пречек** (пре-билл) · **Оплатить** (алоҳида тўлаш!).
- Пастда: **«Разделить все»** (ҳаммасини тенг бўлиш) · **Отмена**.

## 2. La Limonariya — ҲОЗИР
- **ЙЎҚ.** Битта заказ = битта чек. `pos.close`да **multi-tender** (битта чекни нақд+карта аралаш) бор, лекин **split BILL** (позицияларни **алоҳида чекларга** бўлиб **алоҳида ёпиш**) — йўқ. **Объединить** (чекларни бирлаштириш) ҳам йўқ.

## 3. BACKEND spec (`router.ts`)
- **`pos.splitOrder({orderId, splits: [{items:[{orderItemId, qty}]}]})`** — битта order'ни N order'га бўлиш (транзакция):
  - Ҳар split → янги `orders` (бир хил hall/table/waiter, `status='open'`), позициялар (`orderItems`) кўчади/бўлинади (qty бўйича).
  - **Кухня тикетлари** аллақачон юборилган (`kitchenTickets`) — **қайта юборилмайди** (фақат тўлов/чек бўлинади; таом бир марта пиширилган).
  - Ҳар split алоҳида `pos.close` билан ёпилади (алоҳида тўлов).
  - Иерархия: **cashier+** (пул). Аудит: `order_events` (split).
- **`pos.mergeOrders({sourceIds: [...], targetId})`** — 2+ очиқ order'ни битта'га: позициялар `targetId`га кўчади, манбалар `status='merged'/'cancelled'`. cashier+. Аудит: merge.
- ⚠️ Тўлов/списание изчиллиги: split/merge'да `orderPayments`/`stockMovements` фақат ёпилганда ёзилади (мавжуд оқим) — split'дан кейин ҳар чек ўзи ёпилади.

## 4. FRONTEND spec (`Pos.tsx` — янги «Разделить» экрани)
- **3-панел UI:** Основной Чек | Разделить-N панеллар (➕ янги split) | split карточкалар рўйхати.
- Позицияни **босиб кўчириш** (Main ↔ Split). Ҳар split: жой/тип, **Пречек**, **Оплатить** (алоҳида), lock. **«Разделить все»** (тенг бўлиш), **Отмена**.
- **Объединить:** очиқ чекларни танлаб битта'га (модал).

## 5. Приоритет / боғлиқлик
- **P2** — банкет/катта стол (меҳмонлар алоҳида тўлайди) — реал ресторанда кўп.
- Боғлиқ: **Пречек** (§ precheck), **Изменить стол** (кўчириш), станция печати (ҳар split чеки).
