# Пречек (тўловдан олдин счёт) — backend + frontend spec — P1

> Тўловдан **олдин** мижозга бериладиган ҳисоб-счёт (пре-билл). Заказ ЁПИЛМАЙДИ. La Limonariya'да ЙЎҚ. Приоритет: **P1** (официантга кундалик).

## 1. CloPOS'да (жонли кўрилди)
- Заказ toolbar → **🖨 Пречек** → жорий чек ҳолати (позициялар + сумма) **принтерга босилади**, мижоз кўради. Заказ **очиқ қолади** (ёпилмайди).
- Чеки рўйхатида чекда **«Пречек N»** badge (неча марта пречек босилган).
- **«Действия»да** «Печать чека» логланади. **Фискал чек эмас** (расмий эмас, тахминий счёт).

## 2. La Limonariya — ҲОЗИР
- **ЙЎҚ.** Фақат **ёпилгандан кейинги** чек (`Pos.tsx` `Chek` компонент → `window.print()`). Очиқ заказ учун счёт печати йўқ.

## 3. BACKEND spec
- **Схема (кичик):** `orders`га `+ precheckCount int default 0` (ихтиёрий) ёки `order_events`да `print`/`precheck` (§ chek-деystviya-audit) — ким/қачон/неча марта.
- **`pos.precheck({orderId})`:** жорий позициялар + subtotal/service/total ни олади (заказни ЎЗГАРТИРМАЙДИ, ledger'га ёзмайди) → **мост-агентга** (§ станция печати, «Чек» принтер) юборади ёки `window.print`. `precheckCount++` + `order_events` (precheck). Иерархия: официант ўзиникини.

## 4. FRONTEND spec (`Pos.tsx`)
- Заказ toolbar **🖨 Пречек** тугма → жорий чекни босади (мост-агент/`window.print`), заказ **очиқ қолади**. Мавжуд `Chek` компонент қайта ишлатилади (лекин «ПРЕЧЕК / тахминий счёт» сарлавҳа билан, «тўланмаган»).
- «Пречек N» кўрсатиш (неча марта).

## 5. Приоритет / боғлиқлик
- **P1** — официант мижозга ҳисоб беради (тўловдан олдин). **Станция печати мост-агенти** (§ stansiya-pechati-bridge) билан бирга — «Чек» принтерга. Fallback: `window.print`.
- Оддий: заказни ўзгартирмайди, фақат печать + event.
