# Мулти-терминал (Локальные + Мобильные терминалы) + KDS — backend + frontend spec

> Бир нечта қурилма (касса, планшет, официант телефони, кухня дисплейи) реал вақтда синхрон ишлаши. **La Limon веб/PWA бўлгани учун асоси тайёр** (URL очиш кифоя). Етишмайдигани: real-time push + KDS экрани. Приоритет: **P2–P3**.

## 1. CloPOS'да (жонли кўрилди)
- **Локальные терминалы:** бир нечта POS десктоп LAN'да, ID+IP билан рўйхатда: **T3** (`192.168.1.11x`, асосий/актив) · Terminal 1 (ID:1, оффлайн) · T2 (ID:2, оффлайн). Асосий терминал **locter-сервер** (express:8100 + WebSocket) орқали бошқаларни синхронлайди.
- **Мобильные терминалы:** официант телефон/планшетлари — «Выберите IP» билан бириктирилади (ҳозир 0). Асосий терминалга уланиб заказ олади.
- **Мақсад:** мулти-касса, официант мобил заказ, KDS (кухня дисплейи) — реал вақт синхрон.
- **Статус панели (жонли):** асосий терминал T3 (192.168.1.245) · **T2 (192.168.1.12)** · **Terminal 1 (192.168.1.15)** — ҳолат + IP. Алоқа компонентлари: Modem · Интернет · **WS** (WebSocket) · **Syncer** (терминаллараро синхрон) · Clopos (сервер). → § top-bar-navigation §2.2.

## 2. La Limonariya — ҲОЗИР
- **★ Веб/PWA афзаллиги:** ҳар қурилма (касса/планшет/телефон) браузердан **бир хил URL**'ни очади, **бир хил серверга** уланади — **терминал бириктириш/locter-сервер шарт эмас** (CloPOS desktop'да керак эди). Бу — катта соддалик.
- **Синхрон:** ҳозир **refetch/polling** (`Pos.tsx` `openOrders` қайта сўралади; экран очилганда/амалдан кейин).
- **ЙЎҚ:** real-time **push** (WebSocket/SSE — заказ ўзгарса бошқа қурилмада дарров), **KDS экрани** (кухня дисплейи — тикетлар оқими).

## 3. Фарқ / афзаллик
| | CloPOS (desktop) | La Limon (веб/PWA) |
|---|---|---|
| Қурилма қўшиш | Терминал бириктириш + locter-сервер | Шунчаки URL очиш ✅ |
| Синхрон | locter WS (мажбурий) | refetch/polling (real-time push йўқ) |
| KDS (кухня дисплейи) | бор | йўқ |
| Мобил официант | IP бириктириш | браузердан ✅ |

## 4. BACKEND spec (қўшимча)
- **Real-time (ихтиёрий, қулайлик):** Hono **WebSocket/SSE** endpoint — заказ/тикет ўзгарса subscribe қилган қурилмаларга push (floor/order авто янгиланади). Ёки мавжуд refetch'ни қисқа interval + optimistic (соддароқ).
- **KDS учун:** `pos.kitchenQueue({station?})` — очиқ тикетлар станция бўйича, ҳолат билан (янги / тайёрланмоқда / тайёр). Мавжуд `kitchenTickets`/`kitchenTicketItems` тайёр асос.
- KDS ҳолати учун: `kitchenTickets`га `+ status pgEnum('new','cooking','done')` (ёки алоҳида `ticket_status` ledger).

## 5. FRONTEND spec
- **KDS экрани (`Kds.tsx`, янги):** станция бўйича очиқ тикетлар — катта карточкалар (стол, таомлар, вақт таймери), «Тайёр» тугма. Ошхона монитори учун (роль: kitchen/manager). Real-time янгиланади.
- **Real-time:** WS subscribe → `FloorView`/`OrderView` авто янгиланади (polling ўрнига) — official/касса бир вақтда кўради.
- **Мобил:** мавжуд PWA — қўшимча иш йўқ (телефон браузеридан ишлайди). «Add to Home Screen».

## 6. Босқичлар / приоритет
1. **KDS экрани** (`kitchenQueue` + `Kds.tsx`) — ошхона дисплейи, P2–P3, фойдали.
2. **Real-time push** (WebSocket) — P2, қулайлик (ҳозир refetch ишлайди).
3. Терминал бириктириш — **керак эмас** (веб афзаллиги, кўчирмаймиз).

## 7. Хулоса Шерхонга
- La Limon **веб/PWA** — мулти-терминалнинг асоси **тайёр** (URL очиш кифоя). CloPOS'нинг терминал-бириктириш/locter мураккаблиги **керак эмас**.
- Қиймат берадигани: **KDS экрани** (кухня дисплейи) + **real-time push** (WebSocket). Иккови ҳам P2–P3.
