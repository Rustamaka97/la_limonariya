## 2. Қўшимча керак бўлганлар (кичик)

### 2.1. 🔔 Bell-notification-center (frontend)
- **Backend тайёр:** `analytics.signals`/`digest` — аномалиялар, кам қолдиқ, узоқ очиқ стол, нарх сакраши, камомад... (§ computeSignals).
- **Frontend (янги):** юқори панелга **🔔 badge** (янги сигнал сони) → dropdown/панел (Новый/Старый). Мавжуд `Dashboard`/`Analitika` сигналларини шу bell'га чиқариш. Роль: director/manager.
- Приоритет: **P3** (қулайлик; сигналлар Dashboard'да бор).

### 2.2. 🟢 → Статус панели (система соғлиғи монитори) — жонли кўрилди
🟢 индикатор **пассив эмас** — босилса **Статус** модали очилади (тўлиқ health-monitor):
- **Система:** Бренд (ЛИМОНАРИЯ) · Филиал (Main) · **IP** (192.168.1.245) · **Терминал** (T3, ID:3) · версия терминала (p4556) · версия илова (2.4.30) · **диск** (64.79 / 183 GB).
- **Алоқа компонентлари** (яшил/қизил): **Modem · Интернет · WS** (WebSocket) · **Syncer** (синхронизатор; бездействующий=бўш) · **Clopos** (сервер).
- **Терминаллар** (жонли ҳолат + IP): T2 (192.168.1.12) · Terminal 1 (192.168.1.15) — 🟢 онлайн, ҳар бирида «Обновить».
- **Принтерлар** (ҳолат + IP:9100 + **MAC** + станция): § stansiya-pechati-bridge.
- **Frontend (La Limon — янги, P3):** ўхшаш **«Статус/Соғлиқ» экрани** — сервер/WS/принтер/терминал ҳолати, диск, версия. **Мост-агент диагностикаси** учун фойдали (принтер онлайн/оффлайн, WS уланиши). PWA `navigator.onLine` + мост-агент ping.

## 3. «+ Новый заказ» → заказ экрани
- «Новый заказ» → тип танлаш → **заказ/меню экрани**. У ердаги функциялар (**Скидка · Пречек · Разделить/Объединить · Изменить стол/официант · Продажи по порциям**) — **алоҳida order-feature MD'лар** (кейинги навбат).

## 4. Хулоса Шерхонга
- Юқори панель **асосан тайёр** (зал/Новый заказ/Чеки/☰/юзер/logout бор).
- Кичик қўшимча: **🔔 bell-center** (сигналлардан) + **🟢 online индикатор**. Иккови ҳам **P3**.
- Асосий иш — «Новый заказ» ортидаги **заказ экрани функциялари** (алоҳида MD'лар).
