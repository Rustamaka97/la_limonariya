# Скидка (чегирма) — backend + frontend spec — P1

> Заказга/позицияга чегирма. La Limonariya'да ЙЎҚ (фақат сервис +10%). Клиент гуруҳ скидкаси билан боғлиқ (§ Клиенты). Приоритет: **P1**.

## 1. CloPOS'да
- Заказ toolbar → **Скидка** → чегирма чекка (ёки позицияга) қўлланади.
- **Диалог (жонли ТЕСТ ҚИЛИНДИ — реал input):** рақам-панел (1-9, 0, .) билан **ФОИЗ (%)** терасан → **Применить** / **Удалить**. CloPOS натижани **«500.00 so'm (5%)»** деб кўрсатади (яъни киритилган % → so'm ҳисоблаб). ⚠️ **Чегирма — ФОИЗ (%) асосида**, subtotal'дан айирилади (сумма эмас!).
  - **Жонли исбот:** ПОМИДОРЧА (10,000) + **5% скидка** → Скидка 500 so'm → **Итого 11,000 → 10,500** ✅.
  - → schema: **`discountPct int`** (фоиз) асосий; `discountAmount` = ҳисобланган so'm (кэш ёки runtime). Клиент гуруҳ скидкаси ҳам % (`group.discountPct`) — мос.
- **Клиент гуруҳ скидкаси** (§ Клиенты, `group.discountPct`) — мижоз бириктирилса авто.
- **X-отчётда** «Скидка» алоҳида қатор; **«Действия»да** «Добавление скидки / Удаление скидки» логланади.

## 2. La Limonariya — ҲОЗИР
- **ЙЎҚ.** `servicePct` (сервис +10%) бор, чегирма йўқ. `orders`да discount майдони йўқ.

## 3. BACKEND spec
- **Схема:** `orders`га `+ discountPct int default 0` (ФОИЗ — CloPOS каби), `+ discountReason text`. (Ихтиёрий: `discountType` агар келажакда сумма ҳам керак бўлса; ҳозир **фоиз** етарли.)
- **Ҳисоб:** `discount = round(subtotal × discountPct / 100)`; `total = subtotal − discount + service`. Скидка **subtotal**'дан (сервис қолган summadan). Жонли: 10,000 × 5% = 500 → Итого 10,500.
- **Клиент гуруҳ:** заказга мижоз бириктирилса → `group.discountPct` авто қўлланади (§ Клиенты).
- **Молия:** `financeForWindow` — «Скидка» = SUM(discount) (X-отчёт қатори); revenue = ҳақиқий тўланган.
- **Роль/лимит:** скидка бериш — **cashier+**; ёки лимит (официант ≤X%, manager чексиз) — фрод назорати.
- **Аудит:** `order_events` — `add_discount`/`remove_discount` (§ chek-deystviya-audit).

## 4. FRONTEND spec (`Pos.tsx`)
- Заказ toolbar **%Скидка** → modal: **% ёки сумма** + сабаб → subtotal'дан айирилади → Итого янгиланади. Чекда «Скидка: −X» кўрсатилади.
- Роль: официант скидка бера олмаса — тугма яширин/блок (иерархия).

## 5. Приоритет / боғлиқлик
- **P1** — кундалик (мижозга чегирма). **Клиент гуруҳ скидкаси** (§ Клиенты) билан бирга режалаш. X-отчёт (§ Создать отчет) ва аудит (§ Действия) билан боғлиқ.
- ⚠️ Аниқлаштириш: **% ёки сумма** (ёки иккови), позицияга ёки бутун чекка — CloPOS Скидка диалоги скриншоти билан тасдиқланади.
