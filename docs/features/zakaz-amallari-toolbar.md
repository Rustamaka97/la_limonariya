## 2.1. Диалог деталлари (жонли — ҳар бирининг ичи кўрилди)
- **Изменить стол** → **floor пикер** (зал/стол харитаси, банд столлар суммаси билан → янги стол танланса заказ кўчади). «Отмена».
- **Изменить Сотрудник** → **ходим рўйхати** (КАССИР·Асадбек·Аббос·Улмас·Sanjarr·Абрам...) + ҳар бирида «**Применить**» → заказ официанти ўзгаради.
- **Изменить тип продажи** → пикер: **На месте · Доставка · С собой**.
- **Изменить количество гостей** → «Выбор кол-ва гостей» **рақам-панел** (1-9,0,.) + Сохранить.
- **Объединить чеки** → «**Открытые чеки**» рўйхати (Чек #N — зал/стол) → танлаб бирлаштириш.
- **Удалить плату за обслуживание** → **toggle** (диалогсиз — 10% сервисни дарҳол олиб ташлайди).
- **Написать комментарий** → матн майдони + Сохранить/Удалить. **Очистить чек** → позицияларни тозалаш (тасдиқ).
- **Скидка** → % рақам-панел (§ skidka) · **Разделить счёт** → 3-панел split (§ razdelit) · **Добавить клиента** → мижоз пикер (§ Клиенты).

## 3. BACKEND spec (янги амаллар — `router.ts`)
- **Разделить счёт** (`pos.splitOrder`): битта order позицияларини 2+ orderга бўлиш (транзакция, kitchen ticket/тўлов ledger сақланади). Роль: cashier+.
- **Объединить чеки** (`pos.mergeOrders`): 2+ order → битта (позициялар кўчади, эскилар cancelled/merged). Роль: cashier+.
- **Изменить стол** (`pos.moveOrder`): `orders.hallId/tableNo` update (иерархия: официант ўзиникини). servicePct зал бўйича қайта.
- **Изменить Сотрудник** (`pos.reassignWaiter`): `orders.waiterId` update. Роль: **manager+** (официант ўзгартира олмайди).
- **Удалить/қўшиш сервис** (`pos.toggleService`): `orders.servicePct` 0↔зал% (роль: manager+, чунки тушумга таъсир).
- **Очистить чек**: барча order items ўчириш (юборилмаган) — очиқ заказни бўшатиш.
- **Ҳар амал → `order_events`** (§ chek-deystviya-audit) — ким/қачон.

## 4. FRONTEND spec (`Pos.tsx`)
