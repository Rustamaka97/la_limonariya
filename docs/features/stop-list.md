# Стоп-лист (таом тугади) — backend + frontend spec

> Таом/маҳсулот вақтинча тугаса — стоп-листга қўшилади → менюда яширинади/қўшиб бўлмайди (тайёр бўлганда — стопдан олинади). La Limonariya'да ЙЎҚ (фақат `active` бутунлай ўчириш + `stock` қолдиқ). Приоритет: **P2**. Оддий.

## 1. CloPOS'да (жонли кўрилди)
- Панель → **Стоп-лист** — категория-таб (ТАОМЛАР · БАЛИКЛАР · САЛАТЛАР · ИЧИМЛИКЛАР · ШАШЛИКЛАР · СПИРТЛИ · ПРОЧЕЕ · Ширинликлар · Мороженое · Non choy).
- Категориядан таом танлаб **стопга қўшилади** → POS менюда **қўшиб бўлмайди/яширинади**. Тайёр бўлганда — стопдан олинади.
- Ҳозир бўш («Список остановок пуст»).

## 2. La Limonariya — ҲОЗИР
- `products.active` (бутунлай ўчириш — стоп эмас), `stock.onHand` (қолдиқ, лекin меню авто блокламайди).
- **ЙЎҚ:** вақтинчалик **«стоп» флаги** (маҳсулот актив қолади, лекин вақтинча сотилмайди).

## 3. BACKEND spec (кичик)
- **Схема:** `products`га `+ stopped bool default false` (ёки `stoppedUntil timestamp` — авто-тикланиш учун).
- **tRPC:**
  - **`pos.menu`** — `stopped=false` (ва `active=true`) ларни қайтаради (стоп таомлар менюда чиқмайди).
  - **`pos.addItem`** — `stopped` таомни рад этади (BAD_REQUEST «таом стопда»).
  - **`catalog.products.setStopped({id, stopped})`** — стопга қўш/ол (роль: manager/cashier — тез реакция учун).
  - `catalog.products.list`га `stopped` майдонини қайтариш.

## 4. FRONTEND spec
- **Стоп-лист экрани/паneli** (Catalog ёки POS ичида): категория-таб → таомлар toggle («стоп»/«актив»). Тез: POS'дан ҳам официант/касса таомни стопга қўя олсин (тугаса).
- **POS менюси (`Pos.tsx`):** стоп таомлар кўринмайди (ёки **хира + блок** «тугади» белгиси билан — яхшироқ UX).

## 5. Хулоса Шерхонга
- **Оддий:** `products.stopped` флаги + `pos.menu` фильтр + `addItem` рад + toggle UI. ~1 миграция, кичик.
- Опция: `stoppedUntil` (эртага авто-тикланиш) ёки кун охирида авто-тозалаш.
- Приоритет: **P2** (тез қадр — «йўқ таомни сотиб қўйиш» олдини олади).
