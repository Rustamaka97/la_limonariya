# Оплата (Произвести платеж) — backend + frontend spec

> Чекни ёпиш = тўлов усули танлаб пул қабул қилиш. **La Limonariya'да ТАЙЁР — ҳатто CloPOS'дан бойроқ.** Приоритет: — (тайёр). «Платежи» интеграция — алоҳида (P3).

## 1. CloPOS «Произвести платеж» (жонли кўрилди)
- **Способы оплаты:** **Наличными** (нақд) · **Картой** (карта) — 2 усул.
- Рақам-панел (1-9, 0, .) билан **сумма** киритилади → **Подтвердить**.

## 2. La Limonariya — ҲОЗИР (ТАЙЁР, бойроқ)
- **`pos.close`** — **5 усул:** `cash` · `card` · `click` · `payme` · `debt` + **multi-tender** (аралаш тўлов, бир чекда бир нечта усул) + **comp** (текин/ходим) + **қарз** (мижоз мажбурий).
- **Frontend (`Pos.tsx`):** «Тўлов усули» modal — нақд/карта/click/payme/қарз тугмалари + «Аралаш тўлов (бўлиб)» + «Текин».
- **Мажбурий текширув:** тўлов суммаси чек жамига тенг бўлиши шарт (тушум яширилмаслиги учун).

## 4. «Платежи» модули (top-bar — алоҳида)
- Юқори панелдаги **«Платежи»** — эҳтимол **Click/Payme/карта интеграцияси** (авто-сверка, QR тўлов, провайдер API) — оддий кассир тўловидан фарқли. Бу **алоҳида интеграция** (P3): Click/Payme API уланиши, тўлов статуси автотасдиқ.
