## 1. Executive summary

Across the 4 branches there are **34 verified feature/fix entries**. After de-duplication (the same feature appears on multiple branches):

- **~19 entries = already-in-main → SKIP.** Almost all infra/security/finance work Rustam did was independently shipped in our main, usually via a stronger implementation: PIN brute-force guard (`rate-limit.ts`, two-stage), daily backup (`docker/backup.sh`, encrypted), Telegram daily digest (`telegram-digest.ts`), `humo` payment (0033), `ega_oldi` owner-draw (0029), order-removal audit (`voidedItems` 0018), and the entire Smena X/Z model (`tillCounts` + `cashCollections`, 0021/0022).
- **~13 distinct features to port** (several appear on 2–3 branches and collapse into one port each).
- **4 new DB migrations** needed: `tovuq` enum value, assets/asset_movements, vitrina/skewer/expiry, tables.pos_x/pos_y.

**Biggest risks**
1. **Migration-number collisions.** Every branch numbered its migrations 0017/0018/0024 — all of which already exist in our main. All must be renumbered to 0037+ and their `meta/_journal.json`/snapshots regenerated, never copied.
2. **Bundled enum duplicates.** Branch migrations bundle `ALTER TYPE … ADD VALUE 'humo'` / `'ega_oldi'` that main already shipped. Applying them verbatim will make Postgres error on duplicate enum value. These statements must be stripped out of every ported migration.
3. **Heavily-diverged shared files.** `schema.ts`, `router.ts`, `Pos.tsx`, `Shell.tsx`, `Catalog.tsx`, `Ombor.tsx`, `Analitika.tsx`, `index.ts` are touched by main's 0018–0036 work AND by many branch features at once. **No branch patch will `git apply` cleanly** — every one is a by-hand additive merge on named anchors, and each branch hunk entangles 2–4 unrelated features that must be split apart.
4. **`ALTER TYPE … ADD VALUE` cannot be used in the same transaction it's added in** (tovuq) — the runner must commit the enum migration before any seed uses the value.
5. **Regression risk in the POS floor redesign** — the branch's simpler FloorView drops main's shipped heat-map, conflict-sheet, long-press and offline-merge. A wholesale copy silently regresses live features.
