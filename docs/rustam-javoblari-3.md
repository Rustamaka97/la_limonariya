# Ответы Рустама ака — раунд 4 (операционные правила) + архитектурные решения

Дата: 2026-06-13. 27 вопросов из `savollar-rustam-aka.md`. Ниже — ответ → решение → что это значит в программе.

## 🔑 ДВА ГЛАВНЫХ ВЫВОДА (определяют ядро системы)

1. **Программа должна САМА АНАЛИЗИРОВАТЬ, а не только записывать.**
   - Q7: владелец даст ВСЕ прошлые обвалки → **программа сама выводит нормы выхода** по частям и ловит отклонения (нормы НЕ хардкодим).
   - Q25: «гумонларим куп, дастуримиз анализ килиб бериши керак» → у владельца много подозрений на утечки; он хочет, чтобы **система сама находила, где теряются деньги/продукт**.
   - → Нужен модуль **аналитики/аномалий**: авто-нормы из истории, выявление выбросов (перерасход, недовес, странные списания), алерты директору. Это «сердце» продукта, не просто склад.

2. **Цены и рецепты фиксированные (кроме Фарша) → учёт детерминированный.**
   - Q12 фикс. цена, Q11 все рецепты стабильны кроме Фарша, Q13 граммовки есть (тех-карты пришлёт). → продажа блюда → списание по фикс. тех-карте; только Шапок→Фарш остаётся ad-hoc.

## Закупка
- **Q1 — поставщик:** каждый раз РАЗНЫЙ мясник. → НЕ фиксированный список; у прихода поставщик опционален (можно просто «базар» + заметка). Не строим обязательный справочник поставщиков.
- **Q2 — цена:** каждый раз РАЗНАЯ. → у каждого прихода/туши СВОЯ цена за кг; в один день могут быть 2 туши с разными ценами. Себестоимость считается по конкретной туше (не средняя).
- **Q3 — оплата мяснику:** каждый раз по-разному (нал / в долг). → нужен учёт **долга поставщику (кредиторка)**, но опционально (приход может быть «оплачен» или «в долг»).

## Склад / инвентаризация
- **Q4 — склады:** 2 места — **Ошхона музлаткич + Катта музлаткич**. → 2 склада (storage), остаток по каждому.
- **Q5 — инвентаризация:** **складчи каждый день**, часто. → ежедневная инвентаризация, роль «складчи», экран пересчёта.
- **Q6 — расхождение:** складчи пишет причину → **директор подтверждает**. → коррекция остатка = причина + аппрув директора (2 шага).
- **Q9 — Брак/Мусор/Чарви:** Брак = Мусор (одно), Чарви — отдельно; **всё через весы**. → 2 категории потерь (брак/мусор, чарви), обе взвешиваются → баланс веса всегда сходится.

## Обвалка
- **Q7 — нормы:** владелец даст ВСЕ прошлые обвалки → **программа сама выводит норму % по частям** и ловит отклонения (Маруф +13.9% авто-ловится). Нормы динамические, из истории.
- **Q8 — % потерь:** ответит позже (частично выведется авто из истории).

## Производство
- **Q10:** Казон кабоб и т.п. — **большим казаном, партией** (не на заказ). → производство = документ-партия.
- **Q11:** ВСЕ рецепты стабильны, **кроме Фарша**. → фикс. рецепты для всего; Шапок→Фарш — единственный ad-hoc.

## Меню / цена
- **Q12:** цена **фиксированная** (даже мясные 250-260к). → блюдо = фикс. цена-порция, не кг×цена.
- **Q13:** граммовки записаны, **тех-карты пришлёт**. → ждём тех-карты (7 шашлыков + рыба + остальные).

## Зал / банкет
- **Q14 — аванс:** иногда берут, иногда нет (по ситуации); скидки редкие, по человеку. → предоплата банкета ОПЦИОНАЛЬНА; скидка ручная, редкая.
- **Q15 — цена банкета:** каждое блюдо ОТДЕЛЬНО (не на человека); **повар-услуга НЕ отдельно** (внутри заведения). → банкет = обычный заказ по позициям; «Повар услуги» как товар НЕ нужен.

## Касса / смена
- **Q16 — размен:** обдумает подробно. → ОТЛОЖЕНО (cash float на старте смены).
- **Q17 — расхождение кассы:** показать директору как **«камомад» (недостача)** — НЕ вешать на кассира авто. → недостача/излишек видна директору.
- **Q18 — инкассация:** в **сейф**. → конец дня нал → счёт «Сейф» (в Clopos есть Safe-счёт).
- **Q19 — возврат:** кассир делает, **но с разрешения директора**. → refund = действие кассира + аппрув директора.
- **Q20 — отмена после кухни:** да, **список виден директору**. → журнал отмен/правок, виден директору (как оранжевый флаг в Clopos).

## Персонал
- **Q21 — складчи:** нужен сейчас; **цель — систематизировать (купить эл. весы)**. → роль «складчи»; интеграция сетевых весов — реальная цель (не «потом»).
- **Q22 — PIN кассира:** да, нужен **персональный PIN** (сейчас 1 человек, но именные). → именные кассиры, у каждого PIN.

## Финансы / контроль (главное)
- **Q23 — KPI директора:** ВСЕ: **выручка, чистая прибыль, долги, топ-блюдо**. → дашборд директора с этими 4.
- **Q24 — деньги vs прибыль:** **оба**. → показывать и кассу (нал), и реальную прибыль (с учётом долгов + остатка мяса).
- **Q25 — утечки:** много подозрений → **система должна сама анализировать** (см. главный вывод 1).

## Будущее
- **Q26 — филиалы:** да, точно (2-й филиал / центральный цех). → архитектура **мультифилиальная** (Clopos уже venue-based — наследуем).
- **Q27 — лояльность:** намерены давать скидки/бонусы регулярам. → лояльность/скидки — будущая фича.
