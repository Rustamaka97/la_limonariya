## 1. КАЛИТ ТУШУНЧА — CloPOS архитектураси

CloPOS **иккига бўлинган**:

| Қисм | Қаерда | Ичида нима |
|---|---|---|
| **Бизнес-логика** (касса UI, ҳисоб-китоб, меню, ҳисоботлар) | **Серверда** — `https://pos.clopos.com/` | Ҳар сафар интернетдан юкланади (`conf.js: TERMINAL_URL`). Кэш **ўчирилган** (`disable-http-cache`). Компьютерда сақланмайди. |
| **Desktop қобиқ + темир кўприги** | `.exe` (`app.asar`) | Ойна, авто-юклаш, янгиланиш, **принтер, тарози, касса тортмаси, SQLite, мулти-терминал** |

**Хулоса Шерхонга:** CloPOS'нинг веб-коди бизга керак эмас (у сервер, минификация, чужой). Бизга керак бўлгани — **темирга ёндашув** (у `.exe` ичида, очиқ ўқилади) ва **UI/UX ғоялари** (жонли кўрдик). La Limonariya'да ўз касса UI'миз бор — ундан ҳам яхшироқ жойлари бор.

### 1.1. Веб ↔ темир кўприги (`inject.js`) — энг муҳим

CloPOS `pos.clopos.com`ни юклаб, унга `window.electron = {...}` объектини инжект қилади. Веб-илова темирни **шу API орқали** чақиради. Механизм: `window.electron.метод(қиймат)` → `ipcRenderer.send("asynchronous-message",{method,value,id})` → main процесс бажаради → `id` бўйича Promise жавоб қайтади (таймаут билан).

**`window.electron` тўлиқ API (La Limonariya мост-агенти учун ТЗ):**

```

findActivePrinters()           — фаол (ARP + пинг) принтерлар
findPrintersWithPort(port)     — берилган портда

printClientReceipt(data)       — МИЖОЗ ЧЕКИ
printStationsReceipt(data)     — СТАНЦИЯЛАРГА (кухня) — ★ калит

```

## 2. CloPOS DESKTOP-ИНТЕГРАЦИЯ (asar коди)

Ишлатган кутубхоналари (ҳаммаси **очиқ кодли** — биз ҳам легал ишлатсак бўлади):
`escpos` + `escpos-network`, `usb`, `serialport`, `sqlite3`, `express` + `cors` + `ws`, `@network-utils/arp-lookup`, `electron-updater/store`, `auto-launch`.

### 2.1. Печать (`lib/printer.js`)
- **3 тур принтер:** `usb` (класс 0x07), `network` (ESC/POS, **порт 9100**), `windows` (драйвер орқали).
- **Тармоқ принтер топиш:** подсетни (x.x.x.0–255) 9100-портда сканлайди + ARP'дан MAC.
- **Печать усуллари:**
  - `printImage` — чекни **расм (base64)** сифатида босади (шрифт/лого муаммоси йўқ) — escpos raster.
  - `printWithCommands` — хом ESC/POS буфер (network→9100, windows→native).
  - `printHtml` — яширин Electron ойнада HTML'ни рендер қилиб босади.
- **Қўшимча:** `cashdraw()` (касса тортмаси очиш), `cut()` (қоғоз кесиш), `beep()` (сигнал).
- **Станция:** битта заказ бир нечта станция принтерига **алоҳида** босилади (кухня, бар, салат, шашлик...).

### 2.2. Тарози (`lib/scaleWeight/reader.js`)
- `serialport` билан COM-порт очади ({path, baudRate}).
- Тарозидан **хом байтларни** веб'га `scale-weight-data` канали орқали ташлайди — **парслашни ВЕБ қилади** (ҳар тарози ҳар хил протокол).
- Уланиш узилса **авто-қайта уланиш**: backoff [1с, 5с, 15с].

### 2.3. Оффлайн (`lib/sqlite.js`) + Мулти-терминал (`server/`)
- SQLite локал база — интернет йўқда ишлаш.
- `express` (порт **8100**) + WebSocket — бошқа терминаллар (KDS кухня дисплейи, официант станциялари) шу терминалга команда юборади. Асосий терминал "сервер" бўлади, POST `/api` → renderer'га IPC → жавоб.

### 4.9. Директор амал-экранлари (жонли кўрилди)

**Сотрудник** — ходимлар бошқаруви (тўлиқ жадвал):
- Устунлар: Полное имя · Дата создания · **Карта** · Статус. Фильтр: Статус/Должность. «Создать сотрудника».
- Ходимлар: Кассир, Асадбек/Аббос/Улмас/Эзоза/Dior/Сарвар... (Ofitsiant), статус Активен.
- **Карта** = карта (RFID/магнит) билан кириш опцияси (ҳозир «Не добавлено»). La Limonariya'да фақат **PIN** (карта йўқ).

**Архив чеков** — ёпилган чеклар (Закрытые таб): ҳар чек №, вақт, зал•стол, официант, сумма + **тўлов усули** (Наличными...). Табллар: Открытые / Закрытые / **Офлайн архив**. Фильтр: Тип продажи · Сотрудник · Зал · Стол · **Курьер**. La Limonariya'да «Чек қидириш» бор, лекин **Офлайн архив** ва **Курьер** йўқ.

**Возврат** (пул қайтариш — форма ичи кўрилди): «**Найти чек**» — чек рақамини **рақам-панелда** (1-9, 0, .) терасан → чек топилади → возврат. La Limonariya'да `finance.refund` **бор** (лекин orderId бўйича; чек-№ бўйича қидириш йўқ).

**Добавить операцию** (ҳар тип ичи кўрилди) — касса пул ҳаракати журнали. **Ҳар тип бошқа майдон:**
| Тип | Майдон/категориялар |
|---|---|
| **Расход** (чиқим) | Другие расходы · Заработная плata · Бонус · Поставки · Корректировка |
| **Доход** (кирим) | Корректировка · Платеж от поставщиков |
| **Перевод** (кўчириш) | **Баланс:** Касса · Карточный счет · Банковский счет · Сейф |
| **Инкассация** | Сумма + Описание (сейфга йиғиш) |

★ **КАЛИТ:** CloPOS'да **бир нечта пул ҳисоби** бор — **Касса · Карточный счет · Банковский счет · Сейф** — ва улар орасида **Перевод** билан пул кўчирилади (кассадан сейфга, карта ҳисобидан банкка ва ҳ.к.).

- **La Limonariya:** `expenses` (Расход) + `cashCollections` (Инкассация) **бор**. **ЙЎҚ:** Доход операцияси, **Перевод**, ва **кўп ҳисоб модели** (Касса/Карта/Банк/Сейф) — La Limon'да пул implicit (битта `expectedCash`). Бу — молиявий аниқлик учун катта фарқ (§7'га қўшилди).

> Иккала амал-экран (Возврат, Операция) — реал пул амаллари, жонли кассада босиб-тасдиқлаш хавфли. Вазифаси аниқ, La Limonariya'да эквиваленти қисман бор.

### 4.10. Қўшимча модуллар (DOM'дан топилди — CloPOS'нинг кенгайтмалари)
Жонли DOM'да қуйидаги модул/интеграциялар аниқланди (La Limonariya'да ЙЎҚ):
- **QR меню** — мижоз QR-код сканерлаб рақамли менюни кўради (контактсиз меню/заказ).
- **Доставка** + **Курьер** — етказиб бериш + курьер бошқаруви (заказ турида «Доставка», Чеки/Архивда «Курьер» фильтри).
- **Платежи** — тўлов интеграцияси (Click/Payme/карта — эҳтимол автоматик сверка).
- **Центральная Система Заказов** — марказий заказ тизими (агрегаторлар: Yandex/Wolt каби ташқи заказлар интеграцияси эҳтимол).
- **Таймер** — таймер (кухня/заказ вақти назорати).

> Булар келажак босқич учун ғоялар. QR меню ва Доставка/Курьер — реал ресторонга фойдали (La Limonariya менюси аллақачон дизайн қилиняпти — QR меню табиий давоми).

## 6. LA LIMONARIYA ҲОЗИРГИ ҲОЛАТИ

**Стек:** Vite + React 19 + TS (PWA, Tailwind v4) · Hono + tRPC + Drizzle · PostgreSQL 17 · Docker (caddy+api+postgres), OptiPlex `192.168.1.4:8080`.

**Код:** ~10.6k қатор. `apps/api/src/router.ts` (монолит, ~3.6k қ.), `apps/web/src/*.tsx` (14 экран). Ledger-based (append-only), advisory-lock, идемпотентлик — **пухта ишланган**.

**14 экран:** Бошқарув · Аналитика · Молия · Касса(POS) · Чек қидириш · Харид · Обвалка · Инвентаризация · Омбор · Ҳисобот · Таннарх · Каталог · Рецептлар · Ходимлар.

**POS'да бор:** зал/стол харитаси (сетка), меню (ранг-код, қидирув), кухняга юбориш (kitchen ticket, станцияга гуруҳлаган), multi-tender тўлов (нақд/карта/click/payme/қарз), қарз (мижоз мажбурий), текин/comp, возврат, чек+кухня тикети **браузер печати** (`window.print()`, 58/80mm).

**Схемада тайёр (лекин ишлатилмаган):** `stations.printable`, `products.soldByWeight` (тарози учун майдон), kitchen tickets станцияга snapshot.

## 8. КЎЧИРИШ РЕЖАСИ (функция бўйича)

### 8.1. ✅ Иерархия/роль қонунлари — БАЖАРИЛДИ
Рустам ака тасдиқлаган қоида: **официант фақат ўзиникини, заказни фақат касса ёпади.**

**Ўзгарган файллар:**
- `apps/api/src/trpc.ts` — `cashierProcedure` (кассир/менежер/директор), `buyerProcedure` (бозорчи+).
- `apps/api/src/router.ts`:
  - `assertOrderAccess(exec, user, orderId)` helper — официант чужой заказга тегса `FORBIDDEN` («Доступ запрещён»). `pos.order/addItem/sendToKitchen/updateMeta/cancel`'га жойланди.
  - `pos.openOrders` — официант **ҳамма банд столни** кўради (иккита заказ хавфи йўқ), лекин чужой заказ **суммаси null** (яширин).
  - `pos.close` → `cashierProcedure` (официант ёпа олмайди).
  - `obvalka.create`, `purchase.create` → `buyerProcedure` (API тешиги ёпилди).
- `apps/web/src/Pos.tsx` — официантга «Ёпиш» ўрнига «Заказни кассир ёпади»; чужой стол суммаси «банд» деб кўрсатилади.
- **Typecheck:** ✅ иккала пакет тоза. Схема ўзгармади → миграция йўқ.

> Шерхонга: бу тайёр код. Хоҳласа `feature/vitrina-print...` бранчда, review қилиб merge қилиш мумкин.

### 8.2. P1 — Станция печати (мост-агент)
**Муаммо:** La Limonariya браузер веб-илова, `window.print()` фақат **битта** принтерга + диалог очади. Ресторонда эса **5 IP принтер станция бўйича** (§3).

**Ечим — локал мост-агент** (CloPOS'нинг ўзи қилгани):
- Кичик Node.js хизмати кассада ишлайди (`escpos` + `escpos-network`, порт 9100, **CP866**).
- La Limonariya веб `sendToKitchen`/чек босганда → агентга HTTP/WS юборади → агент **керакли станция IP'сига** raw ESC/POS босади (диалогсиз, кесиш/сигнал/касса тортмаси билан).
- Схемада тайёр: `stations` (IP майдони қўшилади), kitchen ticket станцияга гуруҳланган.
- **Йўллар:** (A) алоҳида мост-агент .exe/хизмат (тавсия — веб кодга тегмайди), ёки (B) La Limonariya'ни Electron'га ўраш (CloPOS каби).

**Керакли иш:** `stations` жадвалига `ip`, `width` (58/80), `kind` (кухня/чек/бар) майдонлари; агент API (`POST /print` {stationId, escpos-bytes}); станция маршрутизацияси.

### 8.3. P1 — Пречек (предчек)
Тўловдан олдин счёт. La Limonariya'да осон: мавжуд `Chek` компонентига ўхшаш, лекин **очиқ заказ** учун (ёпмасдан), «Пречек» тугмаси → станция/чек принтерга. Ledger'га ёзилмайди (фақат печать). Warning: пречек ≠ фискал чек.

### 8.4. P2 — Тарози
Мост-агент `serialport` билан тарозидан ўқийди → веб'га узатади (CloPOS каби хом байт + веб парслайди). `products.soldByWeight` майдони тайёр — POS'да «тортиш» оқими қўшилади.

### 8.5. P2 — Стоп-лист
`products`га `stopped: boolean` (ёки муддатли) майдон. Менюда яширинади, POS'да қўшиб бўлмайди. Оддий.

### 8.6. P2 — Курьер/Доставка
Заказ турига «доставка» + курьер (ходим) боғлаш. `orders`га `type` (zal/soboy/dostavka), `courierId`. Молия/аналитикага таъсирини текшириш.

### 8.7. P2 — Касса тортмаси
Мост-агент `cashdraw()` (ESC/POS) — нақд тўловда авто очилади. §8.2 билан бирга келади.

### 8.8. P3 — Қолганлари
- **Битта столда бир нечта чек:** `orders` схемаси кўтаради (стол→кўп order), UI'да «Открытые чеки» модали.
- **Floor фазовий харита:** `tables`га `x,y,w,h` + drag-drop таҳрир.
- **Стол печать индикатори:** kitchen ticket ҳолатидан (юборилган/босилган) стол плиткасида 🖨.
- **Оффлайн (SQLite) / Мулти-терминал (KDS):** катта иш — Electron/агент керак. Кейинги босқич.
