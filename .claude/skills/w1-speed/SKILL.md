## Карта кода

- `apps/web/src/lib/cache.ts` — `swr(key, fetcher, onData)`. **Правило из комментария в файле: только медленно-меняющееся (меню/залы/столы/рецепты/ходимлар). Живые заказы и сток через swr НЕ кэшировать.**

- `apps/web/src/Pos.tsx` — карта зала (46 столов), категории, оплата.
- `apps/web/src/Dashboard.tsx:101`, `Hisobot.tsx`, `Analitika.tsx` — сейчас `⏳`-заглушки → skeleton.

## Правила по пунктам

- **1.1 Optimistic UI** (закрытие чека, перенос стола, обвалка submit): UI отвечает мгновенно, мутация уходит через outbox, при ошибке — откат состояния + понятное сообщение на узбекском. Не ломать opId-идемпотентность и FOR UPDATE-серилизацию на сервере — сервер не трогать вообще.
- **1.2 Skeleton**: один переиспользуемый `<Skeleton />` (animate-pulse, серые блоки по форме будущего контента). Без библиотек.
- **1.3 swr везде**: накрыть все read-пути со «статичными» данными; ключи кэша с версией (`catalog:v1`). Заказы/стол-статусы — НЕТ.
- **1.4 Тач**: тап-зоны ≥ 48px на карте зала; long-press (pointerdown + timer 500 мс, отмена при pointermove) = быстрые действия стола (перенести/гости/примечание).
- **1.5 Вибро**: `if ("vibrate" in navigator)` → addItem `[10]`, отправка на кухню `[20]`, закрытие чека `[30,50,30]`.
