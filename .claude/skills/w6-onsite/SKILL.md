# W6 · На месте 🏪

Это не кодинг-фаза — это визит в La Limonariya. Скилл ведёт три режима: `prep` (подготовка), сам чеклист (распечатать/держать под рукой), `report` (фиксация результатов).

## Режим prep — перед визитом

1. Health: `curl -s https://pos77.lalimonariya.uz/api/health` — зелёный.
2. Свежий бэкап: `ssh sanoaterp 'cd ~/la_limonariya && ./docker/backup.sh'` — убедиться, что файл создан.
3. Проверить, что W2.2 (лого-битмап) и W5.1 (пречек) задеплоены — иначе печатать нечего.
4. Сгенерировать печатный чеклист визита (MD → PDF при просьбе) из блока ниже + взять `docs/RUSTAM-AKA-SAVOLLAR-2026-07-03.md` §5.

## Чеклист на месте

**Принтеры (5 станций, IP 192.168.x.131–.137, CP866):**
- [ ] Каждой станции задать IP через Каталог → принтер (`catalog.setStationIp`); флаг printable.
- [ ] Тест кухонного тикета на КАЖДУЮ станцию (узбекская кириллица читаема — CP866).
- [ ] Чек с лого на кассовом 58/80мм: лого не обрезан (384px/576px), низ «Раҳмат! · la-limonariya.uz · @la_limonariya».
- [ ] Пречек: «*** ПРЕЧЕК ***» печатается, заказ остаётся открытым.

**Offline-канарейка (2 устройства):**
- [ ] Планшет A и B открыты на POS → вырубить Wi-Fi роутер.
- [ ] На A набрать заказ (3–4 блюда), на B открыть другой стол — оба работают оффлайн.
- [ ] Включить Wi-Fi → outbox синхронизировался, дублей нет, конфликт-warning (если был) понятен.
- [ ] Индикатор online/offline виден официанту.

**Обучение официантов (10 минут, сценарий):**
- [ ] Открыть стол → гости → блюда → отправить на кухню → пречек → закрыть у кассира.
- [ ] Показать: длинное нажатие стола (быстрые действия), тёмная тема 🌙 вечером.
- [ ] Кто что МОЖЕТ: официант не закрывает чек (только кассир/менеджер), бекор — только менеджер.

**Ответы владельца (§5 — разблокирует #16/#11/#15):**
- [ ] 5а: метод таннарха — last-price / WAC / FIFO?
- [ ] 5б: правила маршрутизации двух складов в реальном времени?
- [ ] 5в: витрина — FIFO-этикетки партий, как маркируем?

## Режим report — после визита

1. Создать `docs/VISIT-<дата>.md`: что работало / что нет / замечания официантов / фото чеков.
2. Ответы §5 вписать в `docs/RUSTAM-AKA-SAVOLLAR-2026-07-03.md` рядом с вопросами (пометка «Жавоб (дата):»).
3. Каждую проблему → задача (TaskCreate) с файлом/шагом воспроизведения; критичное чинить сразу с места по горячему деплою (`./docker/deploy.sh` с авто-rollback).
4. Коммит доков: `docs: visit report <дата>`.

## Правила

Ничего не деплоить ВО ВРЕМЯ обеда/ужина ресторана — только утром до 11:00. Перед любым горячим фиксом на месте — бэкап уже есть (prep-шаг 2), deploy.sh сам откатит при падении healthcheck.
